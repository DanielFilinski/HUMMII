# Моковые данные для админ-панели

## Текущий статус
В данный момент админ-панель использует **моковые данные** для отображения интерфейса и **моковую аутентификацию**.

## Как переключиться на реальный API

### 1. Отключить моковые данные

1. Откройте файл `src/lib/api/admin-client.ts`
2. Найдите строку:
   ```typescript
   const USE_MOCK_DATA = true;
   ```
3. Измените на:
   ```typescript
   const USE_MOCK_DATA = false;
   ```

### 2. Отключить моковую аутентификацию

1. Откройте файл `src/providers/AuthProvider.tsx`
2. Найдите строку:
   ```typescript
   const USE_MOCK_AUTH = true;
   ```
3. Измените на:
   ```typescript
   const USE_MOCK_AUTH = false;
   ```

## Моковая аутентификация

При включенной моковой аутентификации (`USE_MOCK_AUTH = true`):
- Вы автоматически входите в систему как администратор
- Не требуется вводить логин и пароль
- Все защищенные маршруты доступны
- Пользователь: `admin@hummii.com` (Admin User)

## Доступные моковые данные

### Пользователи (Users)
- **5 тестовых пользователей** с разными ролями (CLIENT, CONTRACTOR, ADMIN)
- Включает заблокированных и незаблокированных пользователей
- Верифицированные и неверифицированные аккаунты

### Статистика платформы (Platform Stats)
- Общее количество пользователей: 125
- Активные пользователи: 89
- Всего заказов: 342
- Верифицированных подрядчиков: 45
- Ожидающих верификации: 2
- Общая выручка: 1,250,000 ₽

### Подрядчики на верификации (Pending Contractors)
- **2 подрядчика** ожидают проверки
- С документами и бизнес-информацией

### Логи аудита (Audit Logs)
- **4 записи** различных действий:
  - Блокировка пользователя
  - Добавление роли
  - Верификация подрядчика
  - Создание пользователя

## Страницы с моковыми данными

✅ **Dashboard** - `/admin/dashboard`
- Отображает статистику платформы
- Графики роста пользователей
- Распределение заказов

✅ **Пользователи** - `/admin/users`
- Список всех пользователей
- Фильтрация по ролям
- Поиск по имени/email

✅ **Модерация** - `/admin/moderation`
- Подрядчики на верификации
- Документы для проверки

✅ **Логи** - `/admin/audit-logs`
- История всех действий администраторов
- Фильтрация по типу действия

## Примечания

- Все даты в моковых данных актуальны (октябрь-ноябрь 2024)
- Данные реалистичны и соответствуют типам TypeScript
- При переключении на реальный API убедитесь, что бэкенд запущен на `http://localhost:3000`
