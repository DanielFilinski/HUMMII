# Техническая спецификация - Платформа Hummii

## Блоки главной страницы

### 1. Хедер

- Логотип
- Главное меню: "Как это работает", "Категории", "Помощь/Поддержка"
- Кнопки действий: Войти / Регистрация / Панель профиля вместо регистрации
- Переключатель ролей: "Я заказчик / Я исполнитель" ✓
  - **Уточнение:** Это переключатель ВИДА интерфейса, а не роли пользователя
  - Один пользователь может быть одновременно клиентом И исполнителем
  - Переключатель меняет отображаемые функции (заказы vs отклики)
- Переключатель языка

### 2. Главная секция / Поисковая строка

- Крупный заголовок с обещанием ценности, например: "Найди проверенного специалиста рядом с тобой"
- Поле поиска по услуге, локации или категории
- CTA-кнопка: "Найти специалиста"
- Фоновая иллюстрация или анимация, показывающая взаимодействие заказчика и исполнителя - параллакс

### 3. Популярные категории

- Сетка иконок или карточек: "Ремонт", "Дизайн", "IT-помощь", "Фото и видео", "Красота", "Обучение"
- Возможность перейти в "Все категории" - кнопка к полному списку категорий

### 4. Топ исполнителей / Проверенные специалисты

- Карусель с карточками исполнителей (аватар, рейтинг, категория, количество заказов)
- Кнопка "Смотреть всех"

### 5. Как это работает

Гайд, не блок. Инфографика из 3–4 шагов:
- Зарегистрируйся
- Размести заказ или предложение
- Получи отклики
- Заключи сделку

(сделать в виде попапов или гайдлайнов)

### 6. Отзывы / Доверие

Позже, пока нет:
- Отзывы клиентов о специалистах
- Возможно, блок с логотипами компаний, которые пользуются платформой
- Элементы социального доказательства (рейтинг, количество заказов, отзывы)

### 7. Преимущества платформы

Почему пользователи должны выбрать именно эту платформу:
- "Безопасные сделки"
- "Проверенные специалисты"
- "Поддержка 24/7"
- "Простая регистрация"

(Иконки + короткие описания)

### 8. CTA-блок (призыв к действию) ✓

Повтор ключевого действия:
- "Найди специалиста уже сегодня" или "Стань исполнителем на нашем сервисе"
- Большая CTA-кнопка

### 9. Футер

- Навигация: категории ✓, поддержка ✓, Политика конфиденциальности, Условия использования

## Профиль клиента (заказчика)

### 1. Основная информация

- Аватар ✓
- Имя / никнейм ✓
- Город/провинция ✓
- Время доступности для связи (указывается при создании заказа)

### 2. Управление заказами

Отдельные вкладки ✓:
- **Активные заказы** — карточки с текущими проектами и статусами
- **Завершённые заказы** — история сделок
- **Черновики** — заказы, которые пользователь начал создавать, но не опубликовал

**Полный жизненный цикл статусов заказа:**
1. `draft` - Черновик (создан, но не опубликован)
2. `published` - Опубликован (ожидает откликов)
3. `in_progress` - Принят исполнителем, работа в процессе
4. `pending_review` - Исполнитель завершил, ожидает проверки клиентом
5. `completed` - Завершён (оплачен, отзывы оставлены)
6. `cancelled` - Отменён
7. `disputed` - Открыт спор

### 3. Отклики исполнителей ✓

- Список откликов с кратким описанием исполнителя, рейтингом, ценой и кнопкой "Принять / Отклонить"
- Возможность фильтровать по цене, рейтингу, времени отклика

### 4. Отзывы и оценки ✓

- Отзывы, которые клиент оставил исполнителям
- Рейтинг клиента (исполнители тоже могут оценивать заказчиков — для прозрачности и доверия)

### 5. Финансы

- История платежей через Stripe
- История транзакций и счетов
- Управление методами оплаты
- **Детали:** См. раздел "Платёжная система" ниже

### 6. Настройки

Всплывающее окно ✓:
- Редактирование профиля (имя, фото, контакты)
- Уведомления (email, push) - **Детали:** См. раздел "Система уведомлений" ниже
- Настройки безопасности (смена пароля, 2FA опционально)
- Настройки приватности — можно скрыть номер телефона, имя, локацию и т.д.
- Управление данными (экспорт, удаление аккаунта - PIPEDA compliance)

### 7. Поддержка ✓

- Чат с техподдержкой - AI-бот, который при необходимости перенаправляет на почту
- Кнопка "Сообщить о проблеме"
- Кнопка "Пожаловаться"

## Пользовательский путь - основные аспекты

### 1. Главная страница

**Цель:** Соединить клиента и исполнителя максимально быстро.

**Блоки:**
- **Верхняя панель:** логотип, кнопки "Войти / Зарегистрироваться", переключатель ролей (Клиент / Исполнитель), язык
- **Главный поиск:**
  - поле для ввода услуги или имени исполнителя
  - карта с ближайшими исполнителями (по категориям: ремонт, доставка, репетиторство и т.п.)
  - фильтры (рейтинг, онлайн, верифицированные, цена по договорённости)
- Популярные категории услуг в виде карточек (ремонт, дизайн, репетиторы, перевозки и т.п.)
- Топ исполнителей
- Преимущества платформы
- CTA: "Создать заявку" или "Найти заказ"
- Футер: контакты, политика, поддержка

### 2. Регистрация / Авторизация

**Варианты входа:**
- Через email + пароль
- Через Google / Apple ID (по желанию)

**После входа:** выбор роли
- Клиент
- Исполнитель

**Примечание:** Пользователи могут переключать роли позже в настройках профиля, чтобы не создавать несколько аккаунтов (например, исполнитель, которому нужно нанять кого-то для услуги из другой категории)

### 3. Верификация аккаунта ✓

**Цель:** Повысить доверие.

**Кто должен верифицироваться:**
- **Обязательно:** Исполнители (предоставляют услуги)
- **Опционально:** Клиенты (для повышения доверия)

**Процесс:**
- Зайти в "Настройки" → "Верификация"
- Загрузить фото документа (ID-карта или водительские права)
- Автоматическая верификация через **Stripe Identity**
- Проверка модератором при необходимости
- После подтверждения — на профиле появляется бейдж "✓ Verified"

**Документы для верификации (Канада):**
- Driver's License
- Passport
- Provincial ID Card

## Основные функции

### 6. Чат / Взаимодействие ✓

**Функционал чата:**
- После заявки клиента или отклика исполнителя — создаётся чат
- **Только текстовые сообщения** (без файлов и медиа)
- Максимум 2000 символов на сообщение
- Индикатор "печатает..."
- Read receipts (прочитано/доставлено)
- Возможность редактирования сообщения в течение 5 минут (с отметкой "изменено")
- История чата хранится постоянно (до удаления аккаунта)
- Чат закрывается через 30 дней после завершения заказа (остаётся только для чтения)

**Автоматическая модерация контента:**
- Блокировка телефонов → заменяется на `***`
- Блокировка email адресов → заменяется на `***`
- Блокировка ссылок на мессенджеры (Telegram, WhatsApp, Viber)
- Блокировка социальных сетей (@instagram, @facebook)
- Фильтр нецензурной лексики

**Действия:**
- Кнопка "Закрыть заказ"
- Кнопка "Оставить отзыв"
- Кнопка "Сообщить о проблеме"
- Кнопка "Пожаловаться на сообщение"

**Детали реализации:** См. `docs/modules/chat.md`

### 7. Рейтинг и отзывы ✓

**Клиенты оценивают исполнителей (4 критерия):**
- Quality of Work (Качество работы) - 1-5★
- Professionalism (Профессионализм) - 1-5★
- Communication (Коммуникация) - 1-5★
- Value for Money (Соотношение цена/качество) - 1-5★

**Исполнители оценивают клиентов (3 критерия):**
- Communication (Чёткость постановки задачи) - 1-5★
- Professionalism (Уважительное отношение) - 1-5★
- Payment (Своевременность оплаты) - 1-5★

**Правила:**
- Оценка обязательна, текстовый комментарий - опционален
- 14 дней на оставление отзыва после завершения заказа
- Обе стороны оценивают одновременно (не видят оценку друг друга до публикации)
- Отзыв можно оставить только один раз, редактирование запрещено
- Право на публичный ответ на отзыв
- ✓ Verified Review - отзыв от реального платного заказа

**Расчёт общего рейтинга:**
- Взвешенный рейтинг: 70% средний рейтинг + 20% опыт + 10% верификация
- Минимальный рейтинг для видимости: 3.0★
- Ниже 3.0★ - профиль скрывается из поиска

**Детали:** См. `docs/modules/rating.md`

### 11. Система споров (Disputes) ✓

**Когда можно открыть спор:**
- Заказ выполнен некачественно
- Заказ не выполнен в срок
- Клиент не оплачивает после завершения
- Нарушение условий заказа

**Процесс разрешения спора:**
1. **Открытие спора** - любая сторона может открыть через кнопку "Сообщить о проблеме"
2. **Загрузка доказательств** - фото, скриншоты переписки, описание проблемы
3. **Заморозка средств** - если деньги были переведены, они замораживаются
4. **Рассмотрение модератором** - администратор изучает обе стороны
5. **Решение:**
   - Полный возврат клиенту
   - Полная оплата исполнителю
   - Частичный возврат / частичная оплата
   - Блокировка нарушителя при серьёзных нарушениях

**Сроки:**
- Модератор рассматривает спор в течение 3-5 рабочих дней
- Обе стороны получают уведомления о решении

**Статистика:**
- Количество споров учитывается в рейтинге пользователя
- При множественных спорах - проверка аккаунта

### 12. Система уведомлений ✓

**Каналы доставки:**
- **In-App** - тосты и бейдж на иконке уведомлений (когда пользователь онлайн)
- **Email** - через OneSignal (когда пользователь оффлайн или высокий приоритет)
- **Push** - веб и мобильные push-уведомления через OneSignal

**Приоритеты уведомлений:**

**HIGH PRIORITY (всегда отправляются):**
- Новый отклик на заказ
- Заказ принят/отклонён
- Заказ завершён
- Платёж получен
- Спор открыт
- Изменения безопасности (смена пароля, новый вход)

**MEDIUM PRIORITY:**
- Новое сообщение в чате
- Изменение статуса заказа
- Новый отзыв
- Новый заказ в вашей категории (для исполнителей)

**LOW PRIORITY (email digest):**
- Просмотры профиля (еженедельная сводка)
- Напоминание завершить профиль
- Скоро истекает подписка

**Логика отправки:**
- Если пользователь онлайн → In-App toast + badge
- Если оффлайн → Email (сразу для HIGH, раз в день digest для MEDIUM)
- Push уведомления - если включены в настройках

**Настройки пользователя:**
- Включить/выключить уведомления по типам
- Выбрать каналы (email, push, in-app)
- Частота email (никогда, мгновенно, дневная сводка)

**Технический стек:**
- WebSocket (Socket.io) для in-app
- OneSignal для email и push
- Redis для проверки онлайн-статусов
- PostgreSQL для хранения истории уведомлений

### 13. Геолокация и карты ✓

**Функционал:**
- Поиск исполнителей на карте
- Фильтр по радиусу (5км, 10км, 25км, весь город, провинция)
- Отдельная категория "Онлайн-услуги" (без привязки к локации)

**Определение локации:**
1. **Ручной ввод** (обязательно при регистрации) - город/провинция
2. **Geolocation API** (с согласия пользователя) - для точного поиска
3. **IP-геолокация** (fallback) - только для предложения города

**Конфиденциальность (PIPEDA):**
- Публично показывается только район/город (не точный адрес)
- Fuzzy location на карте (±500м от реальной позиции)
- Точный адрес передаётся только после принятия заказа (в приватном чате)
- Настройки видимости: показывать город, район, или скрыть полностью

**Технологии:**
- Google Maps API (геокодирование, места, расстояния)
- PostGIS (PostgreSQL extension) для поиска по радиусу

**Детали:** См. `docs/api/geolocation.md`

- Уникальные title и description для каждого профиля

**Robots.txt:**
- Разрешена индексация публичных профилей и категорий
- Запрещена индексация: /admin, /api, /dashboard, /chat, /settings, /auth

**Структурированные данные (JSON-LD):**

- AggregateRating для отображения звёзд в поиске Google
- LocalBusiness для исполнителей с физическим адресом

**Производительность:**
- ISR (Incremental Static Regeneration) для профилей - revalidate каждые 24 часа
- next/image для оптимизации изображений
- Redis кэширование для популярных страниц

**Детали:** См. `docs/configs/SEO.md`

### 16. Безопасность и соответствие законодательству ✓

**Аутентификация:**
- JWT с короткой жизнью (15 мин access token, 7 дней refresh token)
- HTTP-only cookies для refresh токенов
- Email верификация (обязательно перед активацией)
- 2FA/MFA (опционально, через TOTP)
- OAuth2.0 (Google, Apple Sign In)

**Защита API:**
- Rate limiting (100 req/min глобально, 5 req/min для auth)
- CORS whitelist
- Helmet.js security headers
- Request payload limits
- DDoS protection

**Данные:**
- Шифрование в покое (PostgreSQL TDE)
- Шифрование в транспорте (TLS 1.3, HTTPS only)
- Полевое шифрование для чувствительных данных
- Маскирование PII в логах

**Валидация:**
- Все входные данные через DTOs (class-validator)
- SQL injection prevention (ORM параметризованные запросы)
- XSS protection (санитизация)
- File upload validation (тип, размер, virus scanning)

**PIPEDA Compliance (Канада):**
- **Право на доступ** - экспорт всех данных (JSON/PDF)
- **Право на исправление** - редактирование профиля
- **Право на удаление** - полное удаление аккаунта с анонимизацией отзывов
- **Право на портируемость** - скачивание данных
- **Прозрачность** - Privacy Policy (EN + FR), Cookie consent
- **Data breach notification** - в течение 72 часов

**Модерация:**
- AI-модерация контента (NSFW, spam)
- Ручная модерация администраторами
- Captcha на регистрации и критичных действиях
- Система жалоб и блокировок

**Детали:** См. `docs/security.md`
